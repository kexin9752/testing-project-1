Finance Trading System

本地服务启动与验证指南

本手册说明如何在 Windows PowerShell 环境下启动后端 FastAPI 服务、前端 Vue 3 开发服务，并完成数据库初始化、接口访问和常见问题排查。

后端技术栈

FastAPI

前端技术栈

Vue 3 + Vite

数据库

SQLite

推荐终端

PowerShell

环境要求

启动前请确认本机已安装以下运行环境，并能在终端中直接执行对应命令。

环境 建议版本 检查命令
Python 3.10 及以上 python --version
Node.js 18 及以上 node --version
npm 9 及以上 npm --version
若 Python 或 Node.js 版本过低，建议先升级环境后再安装依赖，避免依赖包安装失败或开发服务启动异常。

首次启动

首次启动需要分别安装后端依赖、初始化数据库、安装前端依赖。建议打开两个 PowerShell 终端，一个运行后端，一个运行前端。

1. 启动后端服务

进入后端目录，创建虚拟环境并安装 Python 依赖。

cd backend
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt

初始化 SQLite 数据库和演示数据。

python scripts/init_db.py

启动 FastAPI 开发服务。

uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
数据库文件通常会生成在 backend/user_management.db。如果重复执行初始化脚本，演示数据会按脚本逻辑重新写入或更新。

2. 启动前端服务

新开一个 PowerShell 终端，进入前端目录并安装依赖。

cd frontend
npm install

启动 Vite 开发服务。

npm run dev
前端已配置 /api 代理到 http://localhost:8000，因此请先确保后端服务正在运行。

日常启动

依赖安装完成后，后续开发只需要分别启动后端和前端。

后端终端

cd backend
.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

前端终端

cd frontend
npm run dev
如果 8000 或 3000 端口已被占用，需要先关闭占用端口的进程，或临时调整启动端口并同步检查前端代理配置。

访问地址

用途 地址 说明
前端页面 http://localhost:3000 用户登录、仪表盘、用户、部门、交易和报表页面。
后端接口 http://localhost:8000 FastAPI 服务根地址。
Swagger 文档 http://localhost:8000/docs 后端接口调试与参数查看。
健康检查 http://localhost:8000/api/v1/health 用于确认后端服务是否正常响应。

常见问题

前端页面能打开，但接口请求失败

  • 确认后端终端仍在运行，没有启动报错。
  • 打开 http://localhost:8000/api/v1/health 检查健康接口。
  • 确认后端端口为 8000，前端代理默认转发到该端口。

登录失败或没有演示数据

  • 确认已在 backend 目录下执行 python scripts/init_db.py。
  • 确认当前后端运行目录下存在 user_management.db。
  • 如需重置数据，可停止服务后删除数据库文件，再重新执行初始化脚本。

依赖安装失败

  • 先检查 Python、Node.js、npm 版本是否满足要求。
  • Python 依赖安装失败时，可先执行 python -m pip install --upgrade pip。
  • 前端依赖安装异常时，可删除 frontend/node_modules 后重新执行 npm install。

启动检查清单

  1. 后端虚拟环境已激活，依赖已安装。
  2. 已执行 python scripts/init_db.py 初始化数据库。
  3. 后端服务运行在 http://localhost:8000。
  4. 前端依赖已安装，开发服务运行在 http://localhost:3000。
  5. 健康检查接口可正常访问。
  6. 使用默认账号可完成登录。
